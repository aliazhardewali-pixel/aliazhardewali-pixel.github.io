// FEATURED PROJECT

EEG-Controlled Aerodynamic Levitation

A complete brain-computer interface that levitates a ping-pong ball using nothing but focus. A NeuroSky MindWave Mobile 2 headset streams raw EEG data to a Python engine that quantifies mental concentration in real time and drives an ESP32-controlled fan through a 3D-printed nozzle — the harder you focus, the higher the ball floats.

DATA PIPELINE

01 · ACQUIRE

NeuroSky MindWave 2

Dry-electrode EEG headset streams ThinkGear packets over Bluetooth serial at 57.6 kbaud — sync bytes, payload, and checksum verified on every frame.

02 · PROCESS

Python Signal Engine

A threaded parser extracts 8 EEG power bands (Delta → Mid-Gamma), log-normalizes them, detects blinks, and computes a Focus Score from the Beta / (Alpha + Theta) ratio.

03 · TRANSMIT

Serial Bridge

Focus is normalized to 0–100 and streamed to the ESP32 as a single byte over USB serial at 115200 baud — a minimal, latency-free protocol.

04 · ACTUATE

ESP32 + PWM Fan

Firmware maps focus to a 25 kHz PWM duty cycle with a calibrated spin-up floor, driving airflow through a 3D-printed nozzle to levitate the ball.

⟨/⟩ Hardware–Software Connection

The system closes the loop between mind and machine across three layers. The headset handles analog front-end amplification and digitization; Python owns parsing, filtering, and the focus algorithm; the ESP32 owns real-time actuation. Keeping the microcontroller "dumb" — it receives only a pre-computed focus byte — makes the firmware deterministic and the control latency negligible, while all heavy signal processing stays on the host where it can be iterated quickly.

EEG → Bluetooth 57600 → Python → USB 115200 → ESP32 → PWM 25kHz → Fan → 🏐

📊 Live Research Dashboard

A real-time Matplotlib dashboard visualizes the session: a polar radar of all 8 brainwave bands, a rolling band timeline, and a quantitative focus trend with goal threshold. Every packet is logged to CSV for offline analysis, blink events are counted from raw signal spikes, and signal quality is monitored continuously with automatic headset reconnection on dropout.

CONTROL SYSTEMS

Closed-Loop PID Fan Speed Control

A closed-loop RPM controller on an Arduino Nano. A hall-effect sensor measures actual fan speed via interrupt-driven pulse counting, and a tuned PID loop adjusts PWM output to hold a target RPM set by potentiometer or push buttons. A 16×2 I²C LCD shows setpoint vs. actual speed live, with LED indicators for speed zones and anti-windup on the integral term.

PID Control Hall Sensor Interrupts I²C LCD
MICROPROCESSORS

6-Level DC Motor Speed Controller

A dual-input motor controller with six calibrated PWM speed levels driven through a transistor stage. Speed can be set either by potentiometer (mapped into discrete level slices) or by up/down push buttons — the system arbitrates between them and takes the higher level. An LED bar displays the active level, with dedicated ON/OFF status indicators.
